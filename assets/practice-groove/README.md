# Practice Groove — the test song

Original content, generated by this repo's scripts. Free to commit, free to
change, and the only song where the correct answer is known.

| | |
| --- | --- |
| `practice-groove.mid` | The chart. 316 notes, 61.2s, 100 bpm, 4/4, GM channel 10. |
| `practice-groove.flac` | The audio, rendered **from that same chart**. |
| `notes.json` | The note list both were built from. Consumed by the renderer script and the tests. |

## Why it exists

Every real audio + MIDI pair has *unknown* alignment — the MIDI is a
transcription, the audio is a recording, and they share no master (see the
CHANGELOG entry on per-song alignment). That means with a real song you cannot
tell **"gameplay is broken"** apart from **"that pair doesn't line up."**

Here the audio is synthesised from the chart's own note times, sample-accurate.
So the true alignment is known: **offset 0, tempoScale 1**. That makes this an
oracle — for the alignment estimator, and for judging and gameplay once they
exist.

It earned its keep immediately: run against it, the estimator returns
**-6.3ms / scale 1.00000 / confidence 4.78**. Without the `FRAME_LEAD`
correction in `alignment.ts` it reads ~-35ms — a bug no self-consistent test
could have caught.

## Why it's built the way it is

- **All six lanes.** kick 60, snare 50, hi-hat 128 (closed + open), ride 56,
  toms 17, crash 5. The real test song we have (Another One Bites the Dust)
  only uses kick/snare/hi-hat and would never surface a tom/crash/ride bug.
- **Structure, not a loop.** Sections change (hats → ride → open hats), there's
  a 16th-note tom fill at bar 15, and a near-silent break at bar 24. A uniform
  loop is *inherently* ambiguous to align — see the bar-ambiguity test.
- **FLAC, not mp3.** mp3/AAC encoders add ~13-26ms of encoder delay, comparable
  to the ±25ms Perfect window. That would silently destroy the ground truth this
  file exists to provide. FLAC is lossless and sample-exact.
- **Deterministic.** The synth is seeded, so regenerating produces identical
  audio.

## Regenerating

```bash
node scripts/generate-test-song.mjs   # chart + notes.json
python3 scripts/render-test-song.py   # audio (needs numpy + ffmpeg)
npm run test:fixtures                 # decode to PCM for the tests
```

Editing the groove means editing `scripts/generate-test-song.mjs` and running
both — never editing the `.mid` or `.flac` by hand, or they stop being a matched
pair and the oracle silently becomes a lie.
